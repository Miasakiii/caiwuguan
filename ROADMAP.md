# 财务官 - 项目路线图

## 总览

- **总工期**：约 6-8 周（单人开发）
- **MVP 交付**：第 4 周
- **技术栈**：Kotlin 2.0 + Jetpack Compose + Room + Hilt + KSP

---

## Phase 1: 项目基础 (Week 1)

### Task 1.1: 项目初始化

- [ ] 创建 Android 项目，配置 Gradle
- [ ] 配置 Compose + Hilt + Room 依赖
- [ ] 设置项目结构（ui/domain/data/service 分层）
- [ ] 配置 KSP（替代 kapt），确保编译通过
- [ ] 声明 `FOREGROUND_SERVICE_SPECIAL_USE` 权限
- **产出**：可编译的空项目骨架

### Task 1.2: 数据库设计与实现

- [ ] 定义 `BillEntity`（金额使用 `Long` 存储，单位：分）
- [ ] 定义 `BillDao`（CRUD + 统计查询，insert 使用 `OnConflictStrategy.IGNORE`）
- [ ] 创建 `AppDatabase`
- [ ] 编写 Migration 策略
- [ ] 单元测试 DAO 查询
- [ ] `Converters` 中 Gson 复用单例
- **产出**：完整的 Room 数据库层

### Task 1.3: 数据模型与 Repository

- [ ] 定义 Domain 层模型（Bill, Category, MonthlyStats，金额均为 Long/分）
- [ ] 定义 BillRepository 接口
- [ ] 实现 BillRepositoryImpl
- [ ] Hilt Module 配置
- **产出**：Repository 层可注入使用

---

## Phase 2: 通知解析引擎 (Week 2)

### Task 2.1: 通知解析框架

- [ ] 创建 `NotificationParser` 接口
- [ ] 实现 `ParseResult` 密封类（成功/失败/忽略，金额使用 Long/分）
- [ ] 创建 `ParserRegistry`（按包名路由到对应解析器）
- **产出**：可扩展的解析框架

### Task 2.2: 微信支付解析器

- [ ] 收集微信支付通知样本（至少 20 种格式）
- [ ] 实现 `WechatParser`
  - 付款通知解析
  - 收款通知解析
  - 红包通知解析
  - 转账通知解析
- [ ] 提取：金额、商户名、交易类型
- [ ] 单元测试（覆盖所有格式变体）
- **产出**：微信通知解析准确率 ≥ 90%

### Task 2.3: 支付宝解析器

- [ ] 收集支付宝通知样本
- [ ] 实现 `AlipayParser`
  - 付款通知
  - 花呗通知
  - 转账通知
- [ ] 单元测试
- **产出**：支付宝通知解析准确率 ≥ 90%

### Task 2.4: 银行 App 通知解析器

- [ ] 收集银行 App 推送通知格式（工/建/农/中/交/招）
- [ ] 实现 `BankAppParser`
  - 消费通知
  - 收入通知
  - 还款通知
- [ ] 按银行分组处理
- [ ] 单元测试
- **产出**：6 大银行 App 通知解析覆盖

> **注意**：Android 4.4+ 已不允许第三方 App 静默接收 SMS 广播。
> 短信监听已降级为 P2 可选功能，此处仅实现银行 App 通知解析。

### Task 2.5: 智能分类引擎

- [ ] 基于商户名的关键词分类映射表
- [ ] `CategoryClassifier` 实现
  - 规则引擎（关键词匹配）
  - 历史学习（用户修正过的商户记住分类）
  - 支付宝导入时直接使用交易分类字段
- [ ] 分类置信度评分
- [ ] 单元测试
- **产出**：自动分类准确率 ≥ 80%

## Phase 5: 增强功能 (Week 5-6)

### Task 5.1: 账单导入引擎

- [ ] `Importer` 接口定义
- [ ] `WechatCsvImporter`：解析微信账单 CSV
  - UTF-8 BOM 处理
  - 跳过统计摘要行
  - 金额清洗（去 ¥ 符号，转换为分存储）
  - 字段映射：交易时间/交易对方/商品/收·支/金额
- [ ] `AlipayCsvImporter`：解析支付宝账单 CSV
  - GBK 编码自动检测和转码
  - 交易分类字段直接映射 Category
  - 字段映射：交易时间/交易对方/商品说明/收·支/金额
- [ ] `BankCsvImporter`：银行流水 CSV
  - 编码自动检测（UTF-8/GBK/GB2312）
  - 支持收入/支出双列格式
  - 按银行配置列映射
- [ ] `GenericCsvImporter`：通用 CSV
  - 用户自定义列映射 UI
  - 映射规则持久化
- [ ] `Deduplicator`：去重引擎
  - 精确匹配（来源 + 交易单号）
  - 模糊匹配（来源 + 金额 + 时间窗口 ±5min + 商户）
  - 宽松匹配（金额 + 日期 + 商户，跨来源去重）
- [ ] 导入流程 UI
  - 文件选择器（SAF）
  - 自动识别来源
  - 预览（导入 N 条 / 跳过 M 条重复）
  - 确认后批量写入
  - 导入结果报告
- [ ] 单元测试（各格式样本 + 去重逻辑）
- **产出**：微信/支付宝/银行账单一键导入

### Task 5.2: 预算管理

- [ ] 设置月度总预算（分存储）
- [ ] 设置分类预算（分存储）
- [ ] 超支通知提醒

### Task 5.4: 多账本
