# 财务官 MVP 实施计划

## 目标

完成 **财务官** Android App 的 MVP，核心闭环：**通知监听 → 自动解析记账 → 账单列表查看 → 手动补录 + 月度统计**

## 开发方式

- **代码位置**：`F:\su\caiwuguan\`（WSL 路径 `/mnt/f/su/caiwuguan/`）
- **构建**：在 Windows 上用 Android Studio 打开项目，Gradle 构建运行
- **Hermes Agent 负责**：生成全部项目源码文件、Gradle 配置、资源文件

## MVP 范围（Phase 1~4 的精简版）

| 模块 | 包含 | 状态 |
|------|------|------|
| **Phase 1: 项目基础** | Gradle 配置 + 项目骨架 + Room 数据库 + Repository | ✅ 全部 |
| **Phase 2: 解析引擎** | 微信 + 支付宝通知解析器 + 智能分类引擎 | ✅ 核心（微信+支付宝，暂缓银行） |
| **Phase 3: 核心 UI** | 首页/账单列表/手动记账/统计/设置 共4+1 Tab | ✅ 全部 |
| **Phase 4: 后台服务** | NotificationListenerService + 权限引导 | ✅ 核心（暂缓保活优化） |

## 实施步骤

### Step 1: 项目骨架（Gradle + 目录结构）
- root `build.gradle.kts` + `settings.gradle.kts` + `gradle.properties`
- `app/build.gradle.kts`（Compose BOM 2025.04 + Room + Hilt + KSP + Vico）
- `AndroidManifest.xml`（声明通知监听权限、前台服务权限）
- 目录树：`app/src/main/java/com/caiwuguan/` 下各层级
- 资源文件：`res/values/`（strings.xml, colors.xml, themes.xml）

### Step 2: 数据层（Room + Entity + DAO）
- `BillEntity`（金额 Long/分，`OnConflictStrategy.IGNORE`）
- `BillDao`（CRUD + 月度统计查询）
- `AppDatabase`（Room 数据库 + Migration）
- `Converters`（Gson 单例 + 类型转换器）

### Step 3: Domain 层 + Repository
- `Bill` / `Category` / `MonthlyStats` / `TransactionType` Domain 模型
- `BillRepository` 接口 + `BillRepositoryImpl`
- `CategoryClassifier`（关键词映射 + 商户学习）

### Step 4: DI 层（Hilt Modules）
- `DatabaseModule` / `RepositoryModule` / `ParserModule`
- `Application` 类（`@HiltAndroidApp`）

### Step 5: 通知解析引擎
- `NotificationParser` 接口 + `ParseResult` 密封类
- `ParserRegistry`（按包名路由）
- `WechatParser`（付款/收款/红包/转账 4种格式）
- `AlipayParser`（付款/花呗/转账 3种格式）

### Step 6: 核心 UI
- **导航框架**：BottomNavigation（首页/账单/统计/设置）+ Navigation Compose
- **首页**：今日收支卡片 + 本月汇总 + 最近5笔 + FAB
- **账单列表**：按日期分组 + 左滑删除 + 点击编辑 + 月份切换
- **手动记账**：金额输入(元/分转换) + 分类选择 + 商户名 + 日期 + 备注
- **统计页**：月度总览 + 分类饼图(Vico) + 每日趋势折线图
- **设置页**：通知监听权限引导 + 关于页面

### Step 7: 后台服务
- `NotificationListenerService` 实现
- 通知路由到对应 Parser → 解析成功写入 DB
- 去重逻辑 + 异常通知标记
- 权限申请引导流

### Step 8: MainActivity + 入口
- `MainActivity`（`@AndroidEntryPoint`）
- 通知权限检查 + 跳转设置引导
